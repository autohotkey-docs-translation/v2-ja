OnClipboardChange

クリップボードの内容が変更されるたびに、指定された関数が自動的に呼び出されるようにします。

OnClipboardChange Function , AddRemove

パラメータ

機能

型：機能オブジェクト

呼び出す関数オブジェクト。この関数のパラメータと戻り値について、以下に説明します。

AddRemove

型：整数

省略した場合、デフォルトは1（以前に登録された関数の後に呼び出す）です。それ以外の場合は、次のいずれかの番号を指定してください：

  • 1 = 以前に登録した機能の後に機能を呼び出します。
  • -1 = 過去に登録された機能よりも先に機能を呼び出します。
  • 0 = 関数を呼び出しません。

機能

この関数は1つのパラメータを受け取る必要があります：

FunctionName(Type)
Type

型：整数

以下の番号のいずれか1つ：

  • 0 = クリップボードが空っぽになりました。
  • 1 = クリップボードにテキストとして表現できるものが入っている（エクスプローラーウィンドウからコピーされたファイルも含まれる）。
  • 2 = クリップボードには、画像など、まったくテキスト以外のものが入っています。

戻り値

この関数が最後の、あるいは唯一のOnClipboardChange関数である場合、戻り値は無視されます。そうでない場合は、後続の関数が呼び出されないように、0以外の整数を返すことができます。

備考

OnClipboardChange関数がすでに実行されている間にクリップボードが変更された場合、その通知イベントは失われます。これが好ましくない場合は、Criticalを使用します。ただし、この場合、OnClipboardChangeスレッドの実行中に発生する他のスレッド（ホットキーの押下など）もバッファリング/ディファーされます。

スクリプト自身がクリップボードを変更する場合、そのOnClipboardChange関数は通常すぐには実行されません。つまり、クリップボードを変更した関数のすぐ下の関数が事前に実行される可能性があります。即座に実行させたい場合は、クリップボード変更後にSleep20などの短いディレイを使用します。

関連

A_Clipboard, OnExit, OnMessage, CallbackCreate

例

クリップボードの変更ごとにツールチップを短く表示します。

OnClipboardChange ClipChanged

ClipChanged(clip_type) {
    ToolTip "Clipboard data type:" clip_type
    Sleep 1000
    ToolTip  ;チップを切る。
}
