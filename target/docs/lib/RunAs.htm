RunAs

RunおよびRunWaitの後続のすべての使用で使用するユーザー認証情報のセットを指定します。

RunAs User, Password, Domain

パラメータ

User

型：文字列

このパラメータと他のパラメータをすべて省略した場合、RunAs機能はオフになり、Runと RunWaitはデフォルトの動作に戻ります。それ以外の場合は、新しいプロセスが作成されるユーザー名となります。

Password

型：文字列

ユーザーのパスワードです。

Domain

型：文字列

ユーザーのドメイン。ローカルアカウントを使用する場合は、空欄のままにしてください。これでうまくいかない場合は、@YourComputerNameを使用してみてください。

備考

ユーザーアカウント制御（UAC）により制限された権限でスクリプトが実行されている場合、RunAsを使用していても、スクリプトが起動するプログラムも制限されるのが一般的です。プロセスを昇格させるには、代わりにRun *RunAsを使用します。

この関数は、その後のRunおよびRunWaitのすべての使用において、代替ユーザー認証情報を使用する（または使用しない）ことをAutoHotkeyに通知する以外に何も行いません。この機能では、資格証明書は検証されません。

無効なUser、Password、Domainが指定された場合、RunおよびRunWaitは問題を説明するエラーメッセージを表示します（このエラーがTry/Catch文によって捕捉される場合を除きます）。

RunAs機能が有効な間は、RunおよびRunWaitは、ドキュメント、URL、システム動詞を起動することができなくなります。つまり、起動するファイルは実行可能なファイルである必要があります。

この機能を使用するには、「セカンダリログオン」サービスを手動または自動に設定する必要があります（手動に設定した場合は、要求に応じてOSが自動的に起動する必要があります）。

関連

Run, RunWait

例

レジストリエディタを管理者として開く。

RunAs "Administrator", "MyPassword"
Run "RegEdit.exe"
RunAs  ; Reset to normal behavior.
