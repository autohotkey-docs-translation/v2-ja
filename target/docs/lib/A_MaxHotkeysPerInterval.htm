A_MaxHotkeysPerInterval

A_MaxHotkeysPerIntervalおよびA_HotkeyInterval変数は、警告ダイアログが表示される以上のホットキー起動率を制御します。

変数意味デフォルトの値Type
A_MaxHotkeysPerInterval 警告ダイアログをトリガーすることなく、間隔内に押せるホットキーの最大数。 70 Integer
A_HotkeyInterval インターバルの長さをミリ秒単位で指定します。 2000 Integer

備考

これらの組み込み変数は、通常スクリプトの開始時に値を割り当てる必要がありますが（デフォルト値が適切でない場合）、スクリプトはいつでもその値を取得または設定することができます。

もし、不注意でキー入力の無限ループ（Send関数が他のホットキーを誤ってトリガーする）を導入した場合、キーボードイベントの急激な洪水によりコンピュータが応答しなくなる可能性があるため、上記が甘くならないように注意する必要があります。

単純化しすぎた例として、ホットキー^c::Send "^c"はキーストロークの無限ループを生成することになります。これを避けるには、ホットキーの定義に$ prefixを付け（例えば、$^c::）、送信機能でホットキーをトリガーできないようにします。

など、無限ループ以外の手段で限界に達する可能性があります。

  • キーリピート率に対してリミットが低すぎる場合、またはシステムが高負荷の場合にキーリピートします。
  • キーボードやマウスのハードウェアで、一般的なキーリピート速度よりも高速に入力イベントを送信します。例えば、一部のマウスでホイールを左右に傾けると、WheelLeft::やWheelRight::などのホットキーの限界に達するようなイベントが大量に送信されることがあります。

警告ダイアログを完全に無効にするには、A_HotkeyInterval := 0を指定します。

例

警告ダイアログをトリガーすることなく、2000ms以内に最大200個のホットキーを押すことを許可します。

A_HotkeyInterval := 2000  ; 初期値（ミリ秒）です。
A_MaxHotkeysPerInterval := 200
