ControlGetPos

コントロールの位置と大きさを取得します。

ControlGetPos &OutX, &OutY, &OutWidth, &OutHeight, Control, WinTitle, WinText, ExcludeTitle, ExcludeText

パラメータ

&OutX, &OutY

型：VarRef

コントロールの左上隅のX座標とY座標（ピクセル）を格納する出力変数への参照。これらの座標は、ターゲットウィンドウのクライアント領域の左上隅からの相対座標であるため、ControlMoveで使用する座標と同じです。

XまたはYのいずれかが省略された場合、対応する値は格納されない。

&OutWidth, &OutHeight

型：VarRef

Controlの幅と高さ（ピクセル）を格納する出力変数へのリファレンス。省略した場合は、対応する値は保存されません。

Control

型：文字列、整数、またはオブジェクト

コントロールのClassNN、text、HWND、またはHwndプロパティを持つオブジェクトを指します。詳しくは、「制御パラメータ」をご覧ください。

WinTitle

型：文字列、整数、またはオブジェクト

ウィンドウのタイトルなど、対象のウィンドウを特定する条件。WinTitleを参照してください。

WinText

型：文字列

存在する場合、このパラメータは、ターゲットウィンドウの単一のテキスト要素からの部分文字列でなければなりません（付属のWindow Spyユーティリティによって明らかにされます）。DetectHiddenTextがONの場合、隠しテキスト要素が検出されます。

ExcludeTitle

型：文字列

タイトルにこの値が含まれているWindowsは、審査対象外となります。

ExcludeText

型：文字列

テキストにこの値が含まれているウィンドウは、検討の対象外となります。

エラー処理

ウィンドウまたはコントロールが見つからない場合、TargetErrorが投げられます。

備考

コントロールを変更する関数とは異なり、ControlGetPosには自動遅延はありません（SetControlDelayは影響しません）。

ウィンドウのタイトルとテキストは大文字と小文字が区別されます。DetectHiddenWindowsがオンになっていない限り、隠しウィンドウは検出されません。

関連

ControlMove, WinGetPos, Control functions

例

現在マウスカーソルの下にあるコントロールの名前と位置を継続的に更新して表示します。

Loop
{
    Sleep 100
    MouseGetPos ,, &WhichWindow, &WhichControl
    try ControlGetPos &x, &y, &w, &h, WhichControl, WhichWindow
    ToolTip WhichControl "`nX" X "`tY" Y "`nW" W "`t" H
}
